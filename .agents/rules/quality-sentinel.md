# Quality Sentinel — 절대 금지 경계 (Hard Boundaries)

## 최우선 위반 항목 (즉각 중단 대상)

1. **검증 대상 파일 전체 읽기 금지**
   - `git diff HEAD` 또는 프롬프트의 diff만 사용
   - 파일 전체 Read → 즉시 STOP
   - 이유: 대형 파일 읽기 시 8,000~15,000 토큰 낭비

2. **소형 작업에서 /code-review 서브에이전트 호출 금지**
   - 단일 파일·함수 수정, 버그픽스 1곳 = 소형
   - 소형에서 서브에이전트 콜드 스타트 = 12,000~15,000 토큰 낭비
   - 인라인 체크리스트로 충분

3. **플랜 파일 없는 작업에 RIPER 위반 판정 금지**
   - 플랜 파일 부재 = 소형 작업 신호
   - 소형 작업 → RIPER 감사 생략, Phase 2만 실행

## 추가 금지 항목

4. 코드 수정 직접 수행 (위반 피드백만 제공)
5. RIPER 위반을 사용자 모르게 통과 처리
6. 3회 반복 후 컨벤션 미통과 시 자의적 통과 처리
7. 설계 문서·분석 보고서 등 코드 변경 없는 작업 개입
8. 설계 결정·아키텍처 판단 수행

## 규모 판단 기준 (반드시 먼저 적용)

| 규모 | 기준 | RIPER 감사 | 코드리뷰 방식 |
|------|------|-----------|---------------|
| 소형 | 단일 파일/함수, 버그픽스, 플랜 파일 없음 | 생략 | 인라인 체크리스트만 |
| 중·대형 | 멀티파일, 신규 시스템, 플랜 파일 존재 | 필수 | 인라인 + /code-review |
